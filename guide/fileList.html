개발자 웹 접근성 유의사항

  • 기본언어 표시 모든 웹페이지에는 기본언어를 표시해야합니다. lang="ko"
  • 입력양식 1. input 타입 image, hidden, sumit, button, reset을 제외한 모든 입력양식에는 1대1로 대응하는 label이 존재해야 합니다.
    2. label for 값과 id값을 일치시켜주세요. label을 사용하지 못할경우 title 을 사용합니다.
    3. select, textarea 도 포함
  • 사용자 요구에 따른 실행 1. 페이지 로딩시 입력양식으로의 자동 포커스, 입력양식에 값을 입력했을 경우 또는 선택(option, radio, checkbox 등)만으로 새창이 열리거나 포커스가 이동하면 안됩니다.
    2. 오류입력 정정방법의 제공은 제외
    3. 새창으로 열리는 링크에는 title="새창"으로 새창이 열림을 알려야합니다. 특히 window.open등 스크립트로 여는 팝업에는 필수로 넣어야합니다.
  • 오류입력 정정방법의 제공 모든 입력양식에는 오류입력 정정방법을 제공해야합니다. 입력을 잘못 했을 경우 해당 내용을 고지(alert)하고 해당 위치로 포커스를 이동시켜주어야 합니다.
  • 제목 제공 1. 웹페이지 제목은 문서의 내용을 짧게 요약하여 제공해야하며 서로 배타적이어야합니다.
    2. iframe 에는 title 로 내부 문서에 대한 짧은 요약 제목을 달아주시고 빈 프레임일 경우 title="내용없음"으로 넣으면 됩니다.
  • 기타 유의사항 1. iframe으로 불러오는 문서도 웹접근성에 맞아야 합니다. 내부 문서에는 기본언어표시 lang="ko"를 넣어주세요
    2. ajax 사용시 초점을 잃지 않게 해야하며 실시간으로 변하는 컨텐츠의 경우 기능버튼에 title 속성으로 다음 데이터가 변경됨을 고지해야 합니다.
    3. 레이어팝업 사용시에 순차적으로 키보드 이용이 가능하도록 컨텐츠의 논리적인 구조에 위치해야 합니다.
